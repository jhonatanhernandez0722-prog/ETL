PROBLEMA IDENTIFICADO
─────────────────────────────────────────────────────────────────────────────

El sistema estaba usando un mapeo DESTRUCTIVO de campos:
- Nombres largos del CSV (presion_sistolica, saturacion_oxigeno, etc.)
- Se convertían a nombres cortos (ps, spo2, etc.)
- ELIMINABAN los campos originales

Resultado:
✗ saturacion_oxigeno desaparecía → solo quedaba spo2
✗ Campos faltantes mostraban 0 en lugar de "Sin dato"
✗ No se accedía a los datos originales
✗ Incertidumbre: ¿Es 0 un valor válido o dato faltante?


SOLUCIÓN IMPLEMENTADA
─────────────────────────────────────────────────────────────────────────────

CAMBIO 1: NormalizadorCampos.normalizar() - Preservación Completa
───────────────────────────────────────────────────────────────────

ANTES (Destructivo):
  normalizado[corto] = normalizado[largo];
  delete normalizado[largo];  // ← ELIMINABA el original

AHORA (Preservación):
  normalizado[corto] = normalizado[largo];
  // NO eliminar nunca - mantener ambos nombres

Resultado:
✓ presion_sistolica = 138 (original)
✓ ps = 138 (alias derivado)
✓ Ambos están disponibles simultáneamente
✓ saturacion_oxigeno = 98 Y spo2 = 98 (no se elimina nada)


CAMBIO 2: Nuevos Métodos en NormalizadorCampos
───────────────────────────────────────────────────────────────────

obtenerNumero(registro, ...nombresAlternos)
- Intenta acceder a MÚLTIPLES nombres en orden
- Ejemplo: obtenerNumero(reg, 'frecuencia_cardiaca', 'fc')
- Devuelve null si no existe (no 0)
- Resultado: Distingue "dato faltante" de "cero válido"

obtenerValor(registro, ...nombresAlternos)
- Igual pero para valores de texto

obtenerBooleano(registro, ...nombresAlternos)
- Para valores booleanos (fumador, consumo_alcohol)


CAMBIO 3: completarImportacion() - Acceso Inteligente
───────────────────────────────────────────────────────────────────

ANTES:
  const ps = parseFloat(registro.ps) || 0;  // NaN || 0 = 0 (incorrecto!)

AHORA:
  const ps = norm.obtenerNumero(registro, 'presion_sistolica', 'ps') || 0;
  const spo2 = norm.obtenerNumero(registro, 'saturacion_oxigeno', 'spo2');
  const fc = norm.obtenerNumero(registro, 'frecuencia_cardiaca', 'fc');

Objeto paciente preserva TODOS los campos:

  }


CAMBIO 4: renderTable() - Visualización Inteligente
───────────────────────────────────────────────────────────────────

ANTES:
  const spo2 = getSafeValue(p.spo2, '—');  // Solo buscaba spo2

AHORA:
  const spo2 = p.saturacion_oxigeno !== null ? p.saturacion_oxigeno : p.spo2;
  
  formatNumber(val, unit, defaultText='—'):
  - null/undefined → defaultText (no 0)
  - 0 → '0' (valor válido)
  - NaN → defaultText

Resultado en tabla:
✓ PS: 138 (no undefined)
✓ PD: 88 (no undefined)
✓ SPO2: 98% (no undefined)
✓ FC: — (si falta, no 0)
✓ Diagnóstico: "Hipertensión Stage 2" (no undefined)
